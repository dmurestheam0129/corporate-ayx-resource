# corporate-ayx-resource

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态资源的形式存储，不针对任何单个域名或具体网站，主要作为通用型 HTML 页面集合进行管理和维护。

## 目录说明

```
corporate-ayx-resource/
├── pages/           # 存放各个独立 HTML 页面
├── assets/          # 页面引用的静态资源（如图片、样式、脚本等）
├── docs/            # 相关文档或说明
└── README.md        # 本文件
```

- **pages/**：每个 HTML 页面独立存放，文件名应清晰反映页面用途。
- **assets/**：按需组织子目录，例如 `css/`、`js/`、`images/` 等。
- **docs/**：可包含使用说明、版本记录或其他辅助文档。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，不依赖后端服务。
- 页面之间相互独立，无强制关联关系。
- 归档时请确保页面内引用的资源路径正确，建议使用相对路径。
- 提交前请检查页面是否能直接通过浏览器打开，并正常显示。

## 维护说明

- 本仓库由维护人员定期更新，新增页面或修正现有页面。
- 欢迎提交 Issue 或 Pull Request 以协助改进。
- 提交新页面时，请遵循以下基本规范：
  - 页面文件放置在 `pages/` 目录下。
  - 如有配套资源，请放入 `assets/` 对应子目录。
  - 确保页面无外部依赖或已内联必要资源。
- 不建议在页面中包含跟踪代码、广告或任何形式的第三方服务调用。
